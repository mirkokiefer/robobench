# How robobench detects, probes, and servos

This is the guided tour. Every image in this doc is regenerated by a script
in `scripts/` from real frames pulled off `pi5b-node.daslab.dev`.

---

## 1. Detection — finding the gripper tip and the Pi PCB

Source: `robobench/detect.py` · Visualizer: `scripts/visualize_detection.py`

Run it on any frame to dump 6 numbered panels and a combined grid:

```bash
python scripts/visualize_detection.py examples/snapshot_demo.jpg \
    --out docs/detection
```

![detection grid](detection/detection_grid.jpg)

The pipeline:

| step | what it does | params |
|---|---|---|
| **1. input** | one JPEG from `/api/stream` (cam1) | 960×720 |
| **2. green HSV mask** | `cv2.inRange((35,60,30),(85,255,200))` then open+close, take largest contour, centroid → **TARGET** (Pi PCB) | open 3×3, close 9×9 |
| **3. red HSV mask** | wraparound red `(0..10) ∪ (170..180)` then close. Right 45 % of the frame is zeroed so the *red Pi box* doesn't outvote the *red wires*. Largest contour centroid = wires | close 7×7, suppress right side |
| **4. wires centroid** | the centroid alone projected back onto the input — sanity check |
| **5. dark-arm PCA** | crop a band around the wires, mask pixels with grey ≤ 70, run PCA on the mask. The first eigenvector points along the gripper. Forced to point in `+u` so the tip is downstream | band 560×320, dark threshold 70 |
| **6. result** | extrapolate `extrap_px=200` along that axis from the wires → **TIP**. Draw error vector `(TARGET − TIP)` | — |

Everything except step 5 is a vanilla HSV-and-contour pipeline. Step 5 is
the trick that turns "I know where the wires are" into "I know where the
fingertips are" *without* any URDF or kinematics.

If you swap to a different scene, only step 2 (target) and possibly step 3
(gripper) need new colour ranges. Steps 5–6 stay the same.

## 2. Probe — building the image Jacobian

Source: `robobench/probe.py`

For each of the 3 active joints (`shoulder_pan`, `shoulder_lift`, `elbow_flex`):

```text
read state           → q0
POST joint = q0+δ    → settle 1.2 s → snap → detect tip → p+
POST joint = q0−δ    → settle 1.2 s → snap → detect tip → p−
POST joint = q0      → return to start

J[:, i] = (p+ − p−) / (2·δ)        (units: px / deg)
```

Result on the demo run (`δ=1.5°`):

```
J = [[ -3.67   13.00   8.33]      du/dq  (column = pan, lift, elbow)
     [  7.33    1.00   0.33]]     dv/dq
```

Pan is dominantly vertical pixel motion (camera angle), lift and elbow are
dominantly horizontal — so pan + lift form an almost-orthogonal basis in
image space and the pseudoinverse is well-conditioned. Elbow is partially
redundant, which is fine — `J⁺` handles it.

## 3. Servo — closed loop

Source: `robobench/servo.py`

Each iteration:

```python
err     = target_px − tip_px               # (du, dv) ∈ ℝ²
dq      = α · J⁺ @ err                     # ∈ ℝ³ deg
dq      = clip(dq, ±max_step)              # per-iter cap
q_new   = clip(q + dq, joint_limits)       # safety
POST every changed joint
sleep settle
re-snap, re-detect → loop
```

Stops on `|err| < tol_px`, max iterations, joint clamp violation, or
detector failure. Detector failure leaves the arm exactly where it is.

## 4. Producing the docs images

Three reusable scripts. They take real frames in, write images out — that's
exactly how `docs/` was generated.

### Trajectory strip
```bash
python scripts/make_trajectory_strip.py /tmp/cams/servo \
    --out docs/servo_trajectory.jpg --cols 4 --tile 480x360
```
Reads `iter*.jpg` and (optionally) `history.json` from a servo output dir.

### Diff visualizations
```bash
python scripts/make_diff_images.py BEFORE.jpg AFTER.jpg \
    --out docs --prefix servo \
    --label-before "BEFORE  iter 0" --label-after "AFTER   iter 6"
```
Produces `<prefix>_diff_heat.jpg`, `<prefix>_diff_anaglyph.jpg`, and
`<prefix>_before_after.jpg`.

The heat view amplifies the absolute per-pixel max-channel delta, thresholds
out JPEG/sensor noise, and renders motion as red over a darkened base. The
anaglyph view collapses each frame to greyscale and stuffs them into the
green+blue (cyan) and red channels respectively, so stationary parts overlap
into greys and motion blooms in red/cyan.

### Detection grid
```bash
python scripts/visualize_detection.py path/to/frame.jpg \
    --out docs/detection
```
Drops the 6 individual panels plus `detection_grid.jpg`.

## 5. Files map

```
robobench/                  # the library
  client.py                 # NodeClient: state(), command(), snap_jpeg()
  detect.py                 # detect_target, detect_grip_tip, annotate
  probe.py                  # probe-learn the 2x3 Jacobian
  servo.py                  # closed-loop visual servo
  run.py                    # probe + servo end-to-end
  config.py                 # JOINTS, LIMITS, DEFAULT_NODE

scripts/                    # docs / debugging utilities
  visualize_detection.py    # 6-panel detection pipeline grid
  make_trajectory_strip.py  # servo iter*.jpg → tiled grid
  make_diff_images.py       # heat / anaglyph / side-by-side

docs/
  servo_trajectory.jpg      # ← scripts/make_trajectory_strip.py
  servo_before_after.jpg    # ← scripts/make_diff_images.py
  servo_diff_heat.jpg       # ← scripts/make_diff_images.py
  servo_diff_anaglyph.jpg   # ← scripts/make_diff_images.py
  detect_demo.jpg           # ← python -m robobench.detect
  detection/                # ← scripts/visualize_detection.py
    01_input.jpg
    02_green_mask.jpg
    03_red_mask.jpg
    04_wires.jpg
    05_pca_axis.jpg
    06_final.jpg
    detection_grid.jpg
```
